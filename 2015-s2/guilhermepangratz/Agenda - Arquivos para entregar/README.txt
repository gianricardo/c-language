SOBRE O FUNCIONAMENTO:

A agenda foi criada tentando oferecer um funcionamento simples e amigavel.
Todas as entradas de dados por parte do usuario são flexiveis no sentido de que entradas inesperadas não causam 
erros fatais na execução do programa. Durante a utilização as opções oferecem instruções sobre o que devem 
receber para propiciarem o devido resultado.
Ao abrir o programa dados da agenda serão lidos de um arquivo de texto chamado agenda.txt, que pode armazenar 
dados gravados por uma utilização prévia da agenda. Todos os dados da agenda durante a execução do programa 
estarão armazenados em um vetor de contatos, onde podem ser criados novos contados e contatos existentes 
podem ser alterados e consultados. Para salvar a agenda no arquivo de texto para utilização porterior deve-se utilizar
a opção de saida do programa (9). Se a seção de uso da agenda for finalizada de outra maneira que não seja pela 
opção para sair, todas as edições na agenda serão perdidas.
_____________________________________________________________________________________________________________

SOBRE OS ARQUIVOS:

Para usar a agenda basta compilar os arquivos de codigo fonte agenda.c e funcoes_agenda.c, o que pode ser feito
utilziando o Makefile incluido no diretorio. O Makefile utiliza o gcc para fazer a compilação, e o programa de saida se
chama agenda.
Todas os prototipos de funções possuem comentarios que procuram descrever brevemente o funcionamento da função.
A documentação do projeto foi gerada utilizando a distribuição binaria do doxygen para windows, onde basta selecionar
o diretorio do projeto e seguir passos rapidos para gerar a documentação a partir das anotações feita no cogigo fonte.
